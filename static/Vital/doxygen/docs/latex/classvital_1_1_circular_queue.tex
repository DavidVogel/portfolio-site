\doxysubsection{Detailed Description}
\subsubsection*{template$<$class T$>$\newline
class vital\+::\+Circular\+Queue$<$ T $>$}
A generic circular buffer (FIFO) data structure that allows adding and removing elements efficiently. 

\doxylink{classvital_1_1_circular_queue}{Circular\+Queue} is a template-\/based circular buffer that can store a sequence of elements. It supports operations like push\+\_\+back, pop\+\_\+back, push\+\_\+front, pop\+\_\+front, as well as inserting, removing, and accessing elements by index. By using a circular indexing strategy, it can reuse a fixed-\/size buffer and avoid costly memory allocations once reserved capacity is sufficient.

Template \doxylink{classvital_1_1_parameters}{Parameters}\+:
\begin{DoxyItemize}
\item T\+: The type of elements stored in the queue.
\end{DoxyItemize}

Operations\+:
\begin{DoxyItemize}
\item push\+\_\+back, pop\+\_\+back\+: \doxylink{classvital_1_1_add}{Add} and remove elements from the end.
\item push\+\_\+front, pop\+\_\+front\+: \doxylink{classvital_1_1_add}{Add} and remove elements from the front.
\item remove, remove\+At\+: Remove elements at given positions or by value.
\item at, operator\mbox{[}\mbox{]}\+: Access elements by index.
\item Contains iterator support for iteration through elements.
\end{DoxyItemize}

This queue uses a one-\/element gap strategy to differentiate between full and empty states, thus when constructing with capacity {\ttfamily n}, it effectively stores up to {\ttfamily n-\/1} elements.
